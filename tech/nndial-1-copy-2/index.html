NNDial Notes - 1

This is the Part 1 of notes for understanding NNDIAL which is a Network-based End-to-End Trainable Task-oriented Dialogue System developed by the Cambridge Dialogue System Group. This part focuses on the general structure of NNDIAL System.

These notes are written for learning and understanding the NNDIAL dialogue system. Further information regarding this task-oriented dialogue system can be found here: NNDIAL Github

NNDial is an open source toolkit for building end-to-end trainable task-oriented dialogue models. It is released by Tsung-Hsien (Shawn) Wen from Cambridge Dialogue Systems Group under Apache License 2.0.

Prerequisite

  1. Understanding of the machine learning framework Theano: Theano Tutorial
  2. Basic ideas about the neural networks (especially CNN and LSTM) used in NLP: Deep Learning for Natural Language Processing

1. Train the belief tracker (dialogue stage tracker)

Run the following command from the root directory of project to train the belief tracker first.

python nndial.py -config config/tracker.cfg -mode train

  1. nndial.py is the control interface of the NNDial system.
  2. -config sets the configuration file used by NNDial system.
  3. -mode is the parameter for setting the mode.

NNDial actually supports loading network configurations from the directory model (demo can be found in /model/demo). Otherwise, it will initialize the network with self.initNet().

# loading pretrained model if any
if os.path.isfile(self.modelfile):
    if not opts:  self.loadNet(parser,None)
    else:         self.loadNet(parser,opts.mode)
else: # init network from scrach
    self.initNet(config,opts)
    self.initBackupWeights()

1.1 Theano configuration

The Theano neural network structure is configured in NNDial.ready() which is in the file nn.NNDialogue.

def ready(self):

    # initialise network model
    if self.debug:
        print 'setting network structures using theano variables ...'
    self.model = NNSDS(self.enc, self.dec, self.policy,
        self.trk, self.trkinf, self.trkreq, self.belief, self.trk_enc,
        self.use_snapshot, self.dec_struct, self.vocab_size,
        self.input_hidden, self.output_hidden,
        self.inf_dimensions, self.req_dimensions, self.grad_clip,
        self.learn_mode, len(self.reader.snapshots), self.latent)

    # setput theano variables
    self.model.config_theano()
    if self.debug:
        numofparams, trainable = self.model.numOfParams()
        print '\t\tnumber of parameters : %8d' % numofparams
        print '\t\tnumber of training parameters : %8d' % trainable

self.model is the object of class NNSDS which is defined in /nn/nnsds.py. The dialogue stage tracker is consisted of 4 different parts respectively:

  1. Intent Network / Encoder (/nn/encoder.py): Encodes a user query into a distributed vector representation with semantic information, which then conditions a decoder network to generate each system response. NNDial is equipped with both LSTM and CNN encoder.

    • LSTM: $$\mathbf{z}_t=\mathbf{z}_t^N=\mathbf{LSTM}(w_0^t,w_1^t,…w_N^t)$$
    • CNN: $$\mathbf{z}_t=\mathbf{CNN}(w_0^t,w_1^t,…w_N^t)$$

  2. Belief Tracker (/nn/tracker.py):Maintains a multinomial distribution for each informable slot, and a binary distribution for each requestable slot. NNDial has three informable slots (food, pricerange, area) and six requestable slots (address, phone, postcode plus the three informable slots).

  3. Policy Network (/nn/policy.py):Binds the system modules together and output a single vector representing the system action. Its inputs are comprised of the output from the intent network, the belief state, and the DB truth value vector.

  4. Generation Network / Decoder (/nn/decoder.py):Uses the action vector to condition a language generator to generate template-like sentences token by token based on the language model probabilities as following:

$$P(w_{j+1}^t|wj^t,\mathbf{h}{j-1}^t,\mathbf{o}_t)=\mathbf{LSTM}_j(wj^t,\mathbf{h}{j-1}^t,\mathbf{o}_t)$$
